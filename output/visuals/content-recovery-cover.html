I deleted
3 weeks of content
in one command.

188MB. Slides, carousels, worksheets, blog images —
wiped from every commit in my repo's history.

content-machine — bash
$ git filter-branch --index-filter \
  'git rm -rf --cached assets/' \
  --prune-empty --tag-name-filter cat -- --all
# rewrote 247 commits. assets/ gone from all of history.

Claude recovered it in 3 steps
01
git reflog
Found the pre-filter-branch commit hash.
History rewritten — but HEAD positions still logged.
→ HEAD@{19} = 6d30a69. 186MB restored.
02
git fsck --dangling
2 PDFs still missing — deleted before filter-branch ran.
Surfaced orphaned commits not reachable from any branch.
→ Dangling commit f5d9a38 contained both files.
03
git cat-file blob <hash>
Extracted file contents directly from git's object store
by blob hash. No branch needed.
→ Both PDFs recovered byte-perfect.
