Three layers of defense
01
⬡
Git LFS
Binaries tracked on LFS servers, not in the repo object store. Filter-branch can't rewrite what it can't see.
.gitattributes → assets/**/* filter=lfs
02
⬡
Guard Hook
PreToolUse hook blocks filter-branch, reset --hard, and rm -rf on asset paths before any command runs.
.claude/hooks/guard_destructive.py
03
⬡
Recovery Anchors
Tagged commits after every major push. Tags survive filter-branch — permanent pointers into history that gc won't collect.
git tag recovery-anchor-YYYY-MM-DD
