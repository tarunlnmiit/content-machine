Create a professional worksheet PDF (A4, landscape) for "Build Your Data Story: The Visualization Checklist from Insight to Persuasion"

Layout:
- Header (top 20%): Title + tagline about applying ML to production
- 5 sections in vertical stack, each with:
  - Section number + title (bold, #1-5)
  - Bulleted prompts or template code
  - Space for handwritten notes on right side

Colors: Dark blue (#1E3A8A) headers, light gray background, monospace font for code

Sections:
1. Define Your Data Story's Core Message
   Identify a dataset or analysis you completed recently. In one sentence, write the insight that matters most. Now map it: What's the problem your data solves? What's your key finding? What decision should follow? This is your story's spine—your visualization must support it.

2. Choose Your Plot Type (Because Charts Aren't Neutral)
   Take your insight from above. Sketch how it would look as: (1) a bar chart, (2) a line chart, (3) a scatter plot. For each, note what truth it reveals and what it hides. Which shape tells your story best? Which could mislead? Explain why the right choice matters.

3. Spot and Fix Design Mistakes That Kill Credibility
   Review a visualization you made in the last month. Count the colors (max 5–6?). Check your axis labels (descriptive or vague?). Do you have a cluttered legend? Identify one mistake from the blog that applies to you. Rewrite it—what changes?

4. Code It with Intention: Matplotlib or Seaborn
   Build a simple visualization using either library. Include: descriptive axis labels, intentional color choices (max 5–6), meaningful title. For each design choice, write a comment explaining why you made it. The code should tell the story of your thinking.

5. Test Your Story: Does It Land Without Explanation?
   Share your visualization with someone unfamiliar with the data. Don't explain it. Ask: 'What's the main story?' and 'What question do you have?' If their answer doesn't match your intent, what is the plot hiding or over-emphasizing by accident? How would you fix it?
